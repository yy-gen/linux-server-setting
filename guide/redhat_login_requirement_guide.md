# JBCS 및 JBoss EAP 원본 파일다운로드(로그인) 필수 사유 안내

본 문서는 사내 폐쇄망 인프라 구축의 **Day-1(인터넷 허용 기간)** 환경에서도 JBCS(Red Hat Apache)와 JBoss EAP(상용 WAS) 설치 파일을 서버 터미널에서 스크립트(`wget` 등)로 직접 받지 못하고, **반드시 개인이 브라우저로 로그인하여 수동으로 다운로드 및 업로드해야 하는 이유**를 설명합니다.

---

## 1. 왜 터미널에서 `wget`으로 한 번에 받을 수 없는가?

Red Hat Customer Portal의 상용 소프트웨어(ZIP 파일 등)는 누구나 접근 가능한 공개 링크가 아닙니다. 

*   **구독(Subscription) 검증**: 상용 제품이므로 계정이 돈을 지불한 유효한 상태인지 점검하는 검증 절차(SSO 로그인)가 무조건 선행되어야 합니다.
*   **토큰 기반의 일회성 URL 발급**: 로그인 후 다운로드 버튼을 누르면 그 세션에만 할당된 난수 형태의 토큰 주소가 임시 생성됩니다. 
*   **결과**: 따라서 인증 세션 쿠키가 없는 리눅스 터미널에서 `wget https://access.redhat.com/.../jboss-eap.zip` 명령어를 날리면, ZIP 파일이 아닌 **401(권한 없음) 에러 HTML 페이지**만 받아지게 됩니다. 

---

## 2. 개발/테스트 서버인데 굳이 '상용 버전'을 써야 하는가?

가장 근본적인 의문은 *"어차피 테스트용 개발 서버인데 복잡하게 로그인까지 해서 비싼 상용 버전(JBCS, JBoss EAP)을 받아야 하는가?"* 입니다. **결론부터 말씀드리면, 운영(Prod) 서버가 상용 버전이라면 개발(Dev) 서버도 무조건 상용 버전을 똑같이 구성해야 합니다.**

*   **환경 불일치 추방 (Parity)**: 오픈소스 버전(Apache, WildFly)과 상용 버전(JBCS, EAP)은 핵심 엔진은 같지만, **기본 설정값, 모듈 버전, 그리고 선적용된 보안 패치 수준**이 다릅니다.
*   **리스크 방지**: "개발 서버(WildFly)에서는 에러 없이 잘 돌아가던 코드가, 운영 서버(EAP)에 배포하니 알 수 없는 에러를 뿜어내는 현상"을 원천 차단하기 위함입니다. 철저한 상용 환경 시뮬레이션을 위해 개발 서버에도 상용 ZIP 바이너리를 붓는 것이 DevOps의 기본 원칙입니다.

---

## 3. 개발 서버 상용 라이선스 해결 팁 (무료 구독)

개발망에 상용 JBoss EAP를 구성한다고 해서 비용이 낭비되는 것은 아닙니다.

*   **Red Hat Developer Program**: 개발이나 로컬 테스트 목적으로 상용 소프트웨어를 사용할 수 있도록 권한을 주는 [무료 개발자 계정 프로그램](https://developers.redhat.com/)이 있습니다. 
*   **혜택**: 해당 무료 계정으로 로그인만 하면 운영 서버와 100% 동일한 JBCS 및 JBoss EAP ZIP 파일을 합법적이고 과금 없이 다운로드할 수 있습니다. 

---

## 4. (대안) 로그인이 정 귀찮고 빠른 셋업이 목적이라면?

만약 운영 서버 환경과의 100% 일치가 필수가 아니고, **"단순 애플리케이션 기능 테스트를 위해 당장 빠른 컨테이너 기동이 최우선"** 이라면 아래의 **완전 오픈소스 대체재**를 스크립트에 넣어 한 번에 설치할 수 있습니다 (로그인 불필요).

| 역할 | 상용 버전 (로그인 필수) | 오픈소스 대체재 (로그인 불필요) | 설치 방법 |
| :--- | :--- | :--- | :--- |
| **WEB** | JBCS (Red Hat httpd) | **Apache httpd** | `dnf install httpd` 또는 `httpd` 공식 도커 이미지 땡기기 |
| **WAS** | JBoss EAP 7/8 | **WildFly** | 공식 홈페이지에서 `wget`으로 ZIP 즉시 다운로드 가능 |

**요약:**
완벽한 운영망 사전 검증이 목표라면 **무료 개발자 계정으로 로그인하여 원본 ZIP 파일을 수동 확보**하는 정공법을, 완전 자동화 셋업이 목표라면 **WildFly** 기반으로 방향을 틀어야 합니다.
